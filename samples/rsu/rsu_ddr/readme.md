# Embedding Binaries into an Executable using objcopy

This document explains how to embed binary files (e.g., `.rpd` files) into a C executable using `objcopy`, and how to access them at runtime via linker-generated symbols.

---

## How objcopy Generates Symbols

`objcopy` derives symbol names **directly from the input filename** by replacing every non-alphanumeric character (`.`, `/`, `-`) with an underscore `_`, then wrapping it with a `_binary_` prefix and `_start` / `_end` / `_size` suffixes.

### Naming Rule

```
_binary_<mangled_filename>_start
_binary_<mangled_filename>_end
_binary_<mangled_filename>_size
```

### Examples

| Input Filename       | Generated Symbol Prefix              |
|----------------------|--------------------------------------|
| `app_image.rpd`      | `_binary_app_image_rpd`              |
| `app-image.rpd`      | `_binary_app_image_rpd` (dash → `_`) |
| `images/app.rpd`     | `_binary_images_app_rpd` (slash → `_`) |
| `my.factory.rpd`     | `_binary_my_factory_rpd` (dots → `_`) |

> The filename passed to `objcopy` **is** the symbol name — mangled. You do not explicitly pass the symbol name; it is auto-derived.

### Verify Generated Symbols

```bash
nm app_image_rpd.o

# Expected output:
# 0000000000070000 D _binary_app_image_rpd_end
# 0000000000070000 A _binary_app_image_rpd_size
# 0000000000000000 D _binary_app_image_rpd_start
```

## Generating Object Files with objcopy

Run `objcopy` on the binary file to convert it into a linkable `.o` object file:

```bash
aarch64-linux-gnu-objcopy \
  --input-target=binary \
  --output-target=elf64-littleaarch64 \
  --binary-architecture=aarch64 \
  app_image.rpd \
  app_image_rpd.o
```

### Using Custom Symbol Names (Recommended)

To avoid relying on auto-derived names, use `--redefine-sym` to rename the symbols to something clean and predictable:

```bash
aarch64-linux-gnu-objcopy \
  --input-target=binary \
  --output-target=elf64-littleaarch64 \
  --binary-architecture=aarch64 \
  --redefine-sym _binary_app_image_rpd_start=app_image_start \
  --redefine-sym _binary_app_image_rpd_end=app_image_end \
  app_image.rpd \
  app_image_rpd.o
```

## Embedding Two Binaries

Run `objcopy` separately for each binary — each gets its own distinct symbols and `.o` file:

```bash
# First binary
aarch64-linux-gnu-objcopy \
  --input-target=binary \
  --output-target=elf64-littleaarch64 \
  --binary-architecture=aarch64 \
  --redefine-sym _binary_app_image_rpd_start=app_image_start \
  --redefine-sym _binary_app_image_rpd_end=app_image_end \
  app_image.rpd \
  app_image_rpd.o

# Second binary
aarch64-linux-gnu-objcopy \
  --input-target=binary \
  --output-target=elf64-littleaarch64 \
  --binary-architecture=aarch64 \
  --redefine-sym _binary_factory_image_rpd_start=factory_image_start \
  --redefine-sym _binary_factory_image_rpd_end=factory_image_end \
  factory_image.rpd \
  factory_image_rpd.o
```

## Linking Object Files into the Executable

### Command Line

```bash
aarch64-linux-gnu-gcc \
  main.o \
  app_image_rpd.o \
  factory_image_rpd.o \
  -lrsu \
  -o app_final.elf
```

If the `.o` files are in a local folder:

```bash
aarch64-linux-gnu-gcc \
  main.o \
  ./objects/app_image_rpd.o \
  ./objects/factory_image_rpd.o \
  -lrsu \
  -o app_final.elf
```

### CMake

```cmake
set(RPD_OBJ_DIR ${CMAKE_SOURCE_DIR}/objects)
set(APP_RPD_OBJ     ${RPD_OBJ_DIR}/app_image_rpd.o)
set(FACTORY_RPD_OBJ ${RPD_OBJ_DIR}/factory_image_rpd.o)

add_executable(app_final
    main.c
    ${APP_RPD_OBJ}
    ${FACTORY_RPD_OBJ}
)
target_link_libraries(app_final PRIVATE rsu)
```

Or using `target_link_libraries` with an OBJECT library (cleaner approach):

```cmake
add_library(embedded_rpd OBJECT IMPORTED)
set_target_properties(embedded_rpd PROPERTIES
    IMPORTED_OBJECTS "${APP_RPD_OBJ};${FACTORY_RPD_OBJ}"
)

add_executable(app_final main.c)
target_link_libraries(app_final PRIVATE embedded_rpd rsu)
```

## Accessing Embedded Binaries in C

Declare the symbols generated by `objcopy` as `extern char[]` in your C code:

```c
/* Symbols from objcopy */
extern char app_image_start[];
extern char app_image_end[];

extern char factory_image_start[];
extern char factory_image_end[];

/* Compute size via pointer arithmetic — do NOT use _size symbol directly */
int app_size     = (int)(app_image_end     - app_image_start);
int factory_size = (int)(factory_image_end - factory_image_start);

void *app_buf     = (void *)app_image_start;
void *factory_buf = (void *)factory_image_start;
```

## Prerequisites

> **Important:** The pre-built object files `app_image_rpd.o` and `ssbl_image_bin.o` must be present in the `samples/rsu/rsu_ddr/obj/` directory before building. These are the `objcopy`-generated objects embedding `app2.rpd` and `fip2.bin` respectively. CMake links them directly into the executable — if either file is missing from `obj/` the build will fail.
